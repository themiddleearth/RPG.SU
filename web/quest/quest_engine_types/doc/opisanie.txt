Итак. 
1. Папка Data Bases содержит SQL коды для создания новых таблиц и изменения пары старых, а также данные о предпололжительно нужных новых предметах.
2. Папка Files содержит все новые и модифицированные файлы игры, в каждом каталоге есть файл Описание.txt, описывающий смысл и суть изменений каждого файла.
3. Что мы имеем.
3.1. На данный момент разработано и альфа-протестировано на Анстейбле 11 типов квестов. 
Код = 1. 
Название: Убить большого монстра.
Суть: Для игрока генерируется, в зависимости от его характеристик, специальный бот, явно превышающий его по силе, т.е. игроку понадобиться несколько "заходов", чтобы его убить.
Код = 2. 
Название: Убить нескольких монстров.
Суть: Для игрока генерируется, в зависимости от его характеристик, выбираются некоторые параметры (сила, ловкость и т.п.) и их значения. Игрок должен убить заданное количество ботов с хар-ками не ниже, чем заданные параметры.
Код = 3. 
Название: Набрать опыта.
Суть: Игрок должен набрать заданое количество опыта (пока что источники опыта - бои с ботом или человеком).
Код = 4. 
Название: Набрать победы.
Суть: Игрок должен набрать заданое количество побед в боях с человеком.
Код = 5.
Название: Отнести посылку в город.
Суть: Игрок должен доставить в заданный город посылку (т.е. прибыть туда в срок). В городе ему выдается письмо с подтверждением, при доставке которого выдается награда.
Код = 601. 
Название: Головоломка Судоку.
Суть: Игрок должен решить головоломку типа судоку. Т.е. есть поле 9х9 клеток, в некоторых расположены цифры от 1 до 9. Клетки группируются в 9 квадратов 3х3. Задача - расположить цифры в свободных клетках так, чтобы цифры не повторялись ни в столбце, ни в строке, ни в квадрает 3х3. В нашем варианте цифры заменены цветами.
Код = 7. 
Название: Принести Рар.
Суть: Игрок должен принести к НПЦ Р или ОР предмет, причем не надетый на него. НПЦ не забирает вещь, только "смотрит".
Код = 801. 
Название: Принести Дорогой предмет.
Суть: Игрок должен принести к НПЦ один из предложенных на выбор не Редких предметов, но относительно дорогих. Если принесено несколько вариантов, НПЦ сам решает (рандом), какой именно он заберет.
Код = 802. 
Название: Принести Н недорогих предметов по торговцу и типу.
Суть: Игрок должен принести к НПЦ заданное количество не редких предметов заданного типа, купленных у заданного торговца. Поднятые с земли или купленные на рынке предметы не учитываются.
Код = 803. 
Название: Принести Н недорогих по хаор-кам.
Суть: Игрок должен принести к НПЦ заданное количество не редких предметов повышающих задланные характеристики не менее, чем на заданное значение.
Код = 804. 
Название: Принести оружие по % поломки.
Суть: Игрок должен принести к НПЦ заданное оружие, % прочности которого будет попадать в заданные пределы.
3.2. Имеется админка для быстрого добавления сюжетов для кветов. Суть в том, что в БД хранится код вывода на экран строки, со вклеенными в нее переменными и функциями, возвращающими значение. При выводе сюжета этот код исполняется, и в тескт подставляются данные конкретного квеста. Имеется также база и функция для вставки в текст случайных вводных слов, междометий и обращений, не несущих смысловой нагрузки, но разнообразящих текст. 
3.3. Базы данных для работы движка, а также несколько изображений, к сожалению, их пока мало (см. Файлы)
3.4. Журнал кветов, где отображаются взятые игроком задания, статус их исполнения и некоторая помощь по исполнению.
4. Что мы собираемся иметь.
4.1. Нужна система зависимости времени на квест и награды от типа и параметров квеста, а также зависимости награды от измененного игроком вермени на квест.
4.2. Нужны, наверное, изображения для НПЦ и предметов.
4.3. Нужны непосредственно сюжеты для квестов. Много в этом направлении уже сделал Демон, как он выберется в инет, пришлет результаты.
4.4. Пока не введен вес квестовых предметов.
4.5. Полка не введено ограничение на частоту взятия квестов для игрока.
Суть: 
1) макс. кол-во квестов в сутки = 2 (3?) 
2) мин. время между повторами у одного НПЦ = 1-2 часа
3) Макс. кол-во в неделю = 6-7

quest_engine_sudoku.php - отвечает за работу типа квестов Головоломка Судоку.
quest_engine_tavern.php - отвечает за диалог в таверне города в квесте с транспортировкой посылки в город.
quests_engine_getting.php - отвечает за получение квета у НПЦ.
quests_engine_leaving.php - отвечает за сдачу квеста НПЦ.
quests_engine_lose_combat.php - обработчик проигрыша боя для типа с большим монстром - сохраняет прогресс.
quests_engine_start_combat.php - обработчик начала боя для типа с большим монстром - меняет параметры бота в соответствии с параметрами игрока.
quests_engine_wincheck.php - обработчик победы над ботом. для типа с большим монстром - добавляет персу его голову, для типа с несколькими монстрами - часть их тела.


standart_func.lib.php - несколько функций для вывода сюжетов квестов.
standart_vars.inc.php - унификация квестовых переменных для вывода в сюжетах.
standart_vars_view.inc.php - унификация квестовых переменных для вывода в просмотре сюжета через админку.

init_owner.php - вспомогательный файл.
quests_engine_chek.php - главный файл получения\сдачи кветов.
